  • Cognitive Search in PostgresSQL [code]

    The project included research and implementation of word embedding based queries which can be utelized in PostgresSQL. We used a database containing research papers and a database of movie information. We provided a plugin for PostgresSQL which enables the user to perform queries like this: 

    Select the 10 most similar movies to the movie ’The Godfather’
    Result: ’The Godfather 2’, ’The Godfather 3’, ’Scarface’, ’Good Fellas’…

    For the implementation I used Facebooks fastText library for the training of word embeddings. The main task of this project was trying different pre-processing stratgies since fastText is designed for texts and not for databases as well as finding the best hyper-parameters. Furthermore, I had use the fastText model in PostgresSQL with a reasonable performance. The presentation of results and a working prototype can be found on GitHub
